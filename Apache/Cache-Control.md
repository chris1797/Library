# 웹서버 Header의 Cache-Control

운영서버 배포 중 이슈를 처리하며 웹서버 header 의 Cache-Control에 대해 알게 되었다. 주로 성능 관련해서 사용하면 좋을 것 같다.

HTTP Cache-Control 은 웹브라우저가 기본적으로 수행하는 HTML, CSS, JS, 및 assets 같은 리소스 파일들 캐싱 전략을 컨트롤 하는 세팅이라고 이해하면 될 것 같다. <br/>
프론트단에서 잘만 사용하면 꽤나 좋은 캐싱 전략이 될 것 같았다.

<br/>

max-age
---
: max-age 는 한번 받아온 리소스의 유효기간을 정해주는 것이라 할 수 있겠다. 해당 유효기간 안이라면 브라우저는 별도의 서버 요청 없이 캐싱된 리소스를 사용한다. <br/>
해당 값은 초(s)로 설정된다.

<br/>

no-cache
---
: max-age=0 과 동일한 의미, 캐시는 저장하되 사용할 때마다 서버에 해당 캐싱 리소스가 유효한지 요청을 보낸다.

<br/>

no-store
---
: 캐시를 절대 하면 안되는 리소스에 사용한다. 저장조차 하지 않는다.
